What Changed Since February 18

✅ FIXED — Workflow email branding (company auto-inject)

All workflow emails now automatically pull the company's name, phone, email, address, and website from the database. No hardcoded names. Works for every tenant. Also injects estimate_url so leads get a direct link to view/accept their estimate.

✅ FIXED — Sender name auto-resolution in workflow emails

Email sign-offs now resolve the sender's real name using a 5-step lookup chain: UI input → estimate fields → staff profile → company admin → AI assistant name fallback. YICN Roofing confirmed sending as "Alex" (their AI assistant name).

✅ FIXED — Header profile picture + correct user name

The header avatar now shows the staff profile photo (uploaded in the team settings). Display name now comes from the staff profile's full_name field instead of the auth table, which was showing "Test User". Kevin Stone's photo and name now display correctly.

✅ FIXED — Fake test companies deleted + company delete function implemented

All 13 fake "John Doe's Company" test records (and 1 "Platform Admin's Company") were permanently deleted with full cascade cleanup across all 14 related tables. The platformDeleteCompany backend function is now implemented and working — protected companies (YICN, platform_default, companysync_demo) cannot be deleted.

✅ FIXED — Spanish/English language switcher

The language switcher was broken due to a query key mismatch — useTranslation was listening to translation-lang-check but the save action invalidated translation-company. Fixed: both now use the same key. Also fixed the sort parameter (created_date → created_at) and added immediate localStorage + storage event dispatch so the UI switches language instantly on click without waiting for a refetch.

✅ FIXED — FAB menu and mobile bottom nav display

The floating action button menu items were being cut off on narrow/foldable phones. Fixed overflow, stacking order, and the bottom nav bar now has a max-width and center alignment for wider screens.

✅ FIXED — Google Calendar status detection

The General Settings page was showing "Connected via App Connector" regardless of actual connection status. Now it calls checkUserGoogleCalendarConnection and shows the real status with a "Connect Google Calendar" button when not connected. OAuth errors are now logged with the specific cause and the exact redirect URI that must be registered in Google Cloud Console.

Google Calendar Sync — Current Status & Fix Path

Current Status: Not Connected

The Google Calendar OAuth flow is implemented and working in code. The connection fails because the Google Cloud project needs the redirect URI registered and the Calendar API enabled. This is a one-time configuration in Google Cloud Console — not a code problem.

Outstanding Issues (Ranked by Impact)

🔴 CRITICAL — Zero automated tests

Still no test files in the project. With 185 pages and growing complexity, any shared component change can silently break dozens of features. This is the single biggest operational risk to the platform.

🔴 CRITICAL — Google Calendar not yet connected

Needs the one-time Google Cloud Console configuration described above. Calendar sync is a key feature for scheduling and task reminders.

🟡 HIGH — No global React error boundaries

If any component crashes (null reference in a chart, bad API response), the entire page goes blank. Users see a white screen with no recovery option. Should wrap major sections in Error Boundaries.

🟡 HIGH — Large file sizes — hardest to maintain

AIEstimator.jsx: 8,264 lines. CustomerProfile.jsx: 4,111 lines. InspectionCapture.jsx: 3,883 lines. Tasks.jsx: 3,794 lines. These are impossible to review or debug efficiently and should be split into sub-components.

🟡 HIGH — Offline / PWA support is incomplete

A service worker file exists (public/sw.js) but offline photo capture, data caching, and background sync are not implemented. Field workers on job sites with poor signal cannot use the app offline.

🟡 MEDIUM — Duplicate page files still exist

Invoices, Payments, Settings, and Tasks each have two page files (lowercase and PascalCase). Only one version is being used. The duplicates add confusion and maintenance overhead.

🟡 MEDIUM — Spanish translations are incomplete

The translations.js file has an English and Spanish section, but the Spanish section is much shorter — many keys are missing or untranslated. Pages that use useTranslation will fall back silently. The switcher mechanism is now fixed; the translation content still needs completing.

🟢 LOW — Navigation overlaps between Communication sub-pages

Live Call Dashboard, Communication Hub, and Sarah Workspace's "Live Dashboard" tab all show similar call metrics. Campaign Manager (separate page) and Sarah Workspace's "Campaigns" tab are redundant. Could be consolidated to reduce tenant confusion.

Security & Data Isolation Confirmation

Multi-Tenant Data Isolation — Verified ✅

All data queries in the backend use company_id filtering. Each tenant (YICN Roofing, Stone Roofing, CompanySync demo) can only see their own records. Platform-level super admin functions (like platformDeleteCompany) require platform_role = 'super_admin' verification on every call. Tenant admins cannot access another tenant's customers, estimates, leads, invoices, or communications.

The database currently contains 3 active companies: YICN Roofing, Stone Roofing, and the CompanySync demo account. The 14 fake test companies that were created during development have been permanently deleted with full cascade cleanup.
